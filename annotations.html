Annotation devtools

An in-page batch annotation overlay for iterating on prototypes with a coding agent. Click multiple UI elements, leave a note on each, then copy a single structured payload to paste into your agent — far less round-tripping than one tweak per message.

Enable it

Append ?notes=1 to any prototype URL. The toolbar appears bottom-right; without the flag the prototype renders clean.

http://localhost:4174/api-token-management/?notes=1

  1. Click Inspect. Cursor becomes a crosshair. Hover outlines elements the way DevTools does.
  2. Click an element. A numbered pin marks the spot and a popover opens with the element's tag, its shortest selector, and a textarea.
  3. Type a note → Enter. Saves the annotation. Shift+Enter for a newline. Esc cancels the current popover.
  4. Repeat for as many elements as you want. The counter ticks up. Pins stay on the page so you can see what's already queued.
  5. Hit Copy batch. One structured payload lands on your clipboard. Paste it as a single message to your agent — every change gets executed in one pass.

Example payload

Three queued annotations produce this clipboard text:

1. <button> .btn-primary "Create token" → Move to far left of header 2. <div> .alert-info "Treat tokens like passwords." → Change icon to a key glyph 3. <span> .tab.is-active "API tokens" → Underline should be 3px, not 2px

Selector strategy

Selectors prefer the shortest readable form. The picker walks this order until it lands on something unique:

  1. #idIf the element has an id, that wins.
  2. Unique classThe first class on the element that selects only one node on the page.
  3. tag.classIf a single class isn't unique, combine with the tag.
  4. Parent pathClimb to a parent and prepend its selector to disambiguate.

Notes that don't survive

Annotations clear on reload. The tool is meant for short batches you copy out immediately — not a persistent backlog. If you reload before hitting Copy batch, your queue is gone.

Where it lives

The CSS lives in devtools.css, auto-inlined alongside the three-layer chain (but not part of it — see design.md §7). The JS is part of the universal handler block in example.html; copying that block into a new prototype brings the annotation tool with it.

Adding the tool to existing prototypes is two steps:

  1. Re-run inline.py.From designmd/: python3 inline.py ../features/<name>/index.html. Picks up devtools.css automatically.
  2. Copy the JS block from example.html.Replace the prototype's existing <script> block (or append the annotation IIFE alongside the others).
